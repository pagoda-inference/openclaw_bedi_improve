# Deep Browser 能力设计

## 一、定位与目标

### 1.1 核心定位

Deep Browser 是 OpenClaw 系统级别的浏览器能力增强模块，不是一个独立的 Skill。它的目标是让 OpenClaw 智能体能够**深度理解、智能操作、持续观察、积累记忆**网页系统，从而更好地支持市场调研-报告生成这一核心应用场景。

### 1.2 设计目标

- **从被动抓取到主动理解**：不只是抓取页面内容，而是理解页面结构、语义、交互方式
- **从单次操作到持续观察**：每次操作后观察效果，形成反馈循环
- **从无状态到有记忆**：积累网站操作经验，越用越聪明
- **从高成本到零成本**：借鉴 OpenCLI 思路，一次学习，多次零 LLM 成本执行

## 二、四大核心能力

### 2.1 理解能力（Understand）

理解能力是 Deep Browser 的"眼睛"。当智能体面对一个网页时，需要回答以下问题：

**页面类型识别**
- 这是什么类型的页面？（电商列表、博客文章、搜索结果、文档站、表格数据...）
- 页面的主要目的是什么？（展示信息、收集数据、进行交易...）

**页面结构理解**
- 页面的布局结构是什么？（导航在哪、内容区在哪、分页在哪...）
- 数据是如何组织的？（表格、列表、卡片、JSON...）

**交互元素识别**
- 有哪些可以交互的元素？（可点击的按钮、可输入的表单、可滚动的区域...）
- 这些元素的语义是什么？（搜索按钮、下一页、筛选器...）

**数据结构识别**
- 页面上有哪些数据？
- 数据之间的关系是什么？

**输入**：页面 URL 或当前页面状态
**输出**：结构化的页面描述

### 2.2 操作能力（Operate）

操作能力是 Deep Browser 的"手"。智能体需要对页面执行各种操作：

**基础操作**
- 导航：打开 URL、前进、后退、刷新
- 点击：点击按钮、链接、复选框
- 输入：填写表单、输入文本
- 滚动：向上滚动、向下滚动、滚动到元素
- 等待：等待元素出现、等待页面加载、等待条件满足

**复合操作**
- 翻页：识别分页类型，执行翻页操作
- 搜索：定位搜索框，输入关键词，提交搜索
- 筛选：识别筛选器，选择筛选条件
- 展开：点击"查看更多"、"展开"等按钮

**策略操作**
- 根据目标选择最优操作序列
- 根据页面状态动态调整操作策略
- 处理异常情况（弹窗、验证码、错误提示）

**输入**：操作指令
**输出**：操作结果和页面变化

### 2.3 观察能力（Observe）

观察能力是 Deep Browser 的"反馈机制"。每次操作后，智能体需要观察效果：

**变化检测**
- 页面是否发生了变化？
- 变化了什么？（URL 变化、内容变化、状态变化...）
- 变化是否符合预期？

**效果验证**
- 操作是否成功？
- 是否达到了预期目标？
- 是否产生了副作用？

**状态感知**
- 当前处于什么状态？
- 是否有加载中、处理中等中间状态？
- 是否有错误状态？

**异常识别**
- 是否出现了弹窗？
- 是否出现了验证码？
- 是否出现了错误提示？
- 是否需要登录？

**输入**：预期变化
**输出**：实际变化和状态评估

### 2.4 记忆能力（Remember）

记忆能力是 Deep Browser 的"学习机制"。智能体需要积累经验：

**操作记忆**
- 做了什么操作
- 操作的结果如何
- 操作是否成功

**页面记忆**
- 访问过哪些页面
- 页面的内容是什么
- 页面的结构特征

**模式记忆**
- 某个网站有什么特征
- 如何操作该网站最有效
- 该网站的常见问题和解决方案

**知识记忆**
- 学到了什么可以复用
- 某类网站的通用规律
- 操作技能的抽象模板

**核心价值**：让智能体"越用越聪明"——第一次访问某个网站时需要实时分析，第二次就可以直接使用之前的经验。

## 三、工作循环

Deep Browser 的工作方式是一个持续的循环：

**目标 → 理解 → 规划 → 操作 → 观察 → 记忆 → 判断是否完成**

### 3.1 循环详解

1. **接收目标**：明确当前要完成的任务
2. **理解页面**：分析当前页面的结构、内容、交互元素
3. **规划操作**：根据目标和页面状态，决定下一步操作
4. **执行操作**：执行具体的浏览器操作
5. **观察效果**：检测页面变化，验证操作结果
6. **更新记忆**：记录操作历史、页面特征、学到的经验
7. **判断完成**：判断任务是否完成，是否需要继续循环

### 3.2 关键特点

- **反馈驱动**：每次操作后都有观察和验证，形成闭环
- **记忆积累**：每次循环都会更新记忆，为后续决策提供依据
- **动态调整**：根据观察结果动态调整后续操作策略

## 四、与 OpenCLI 的关系

### 4.1 OpenCLI 核心思想

OpenCLI 提出了一个重要的设计哲学：**编译期智能 vs 运行期智能**

- **传统方案**：每次执行都消耗 LLM，成本高、不稳定
- **OpenCLI 方案**：一次生成适配器（消耗 LLM），后续执行零 LLM 成本、100% 稳定

### 4.2 借鉴思路

Deep Browser 借鉴 OpenCLI 的思路：

**网站模式库**
- 预定义通用布局模式（电商列表、博客文章、文档站等）
- 学习并积累特定网站模式
- 首次访问时用 LLM 分析，生成模式描述
- 后续访问直接使用模式，零 LLM 成本

**操作技能沉淀**
- 将成功的操作序列抽象为"技能模板"
- 首次执行时 LLM 实时决策
- 后续执行直接使用技能模板

**确定性适配器**
- 对于频繁访问的网站，生成 OpenCLI 适配器
- 完全确定性执行，零 LLM 成本

### 4.3 集成方式

Deep Browser 可以：

1. **直接使用 OpenCLI 的浏览器命令**：open、click、type、scroll、wait 等
2. **利用 OpenCLI 的预置适配器**：对于已支持的网站，直接调用适配器
3. **生成新的 OpenCLI 适配器**：对于新网站，生成适配器供后续使用

## 五、记忆系统设计

### 5.1 三层记忆架构

**会话记忆（Session Memory）**
- 生命周期：单次任务执行期间
- 内容：已访问的 URL、已获取的数据片段、当前操作状态
- 存储：内存或临时文件
- 用途：避免重复操作、支持任务恢复

**模式记忆（Pattern Memory）**
- 生命周期：跨会话持久化
- 内容：网站布局模式、操作技能模板、成功/失败经验
- 存储：持久化文件
- 用途：加速后续访问、复用成功经验

**知识记忆（Knowledge Memory）**
- 生命周期：长期持久化
- 内容：网站特征知识、数据结构知识、最佳实践
- 存储：持久化文件或知识库
- 用途：泛化到新网站、提升整体能力

### 5.2 记忆内容示例

**网站模式记忆**
```
网站：amazon.com
类型：电商
布局模式：ecommerce-grid
分页类型：click-based
搜索框：#twotabsearchtextbox
商品列表：[data-component-type="s-search-result"]
下一页按钮：.s-pagination-next
注意事项：价格可能延迟加载，需要等待
```

**操作技能记忆**
```
技能：翻页
适用场景：数字分页
操作序列：
1. 检测下一页按钮是否存在
2. 点击下一页按钮
3. 等待页面加载完成
4. 验证页码已更新
成功标志：URL 变化、新内容出现
失败标志：按钮消失、无新内容
```

## 六、完成度评估机制

### 6.1 多维度评估

Deep Browser 需要自主判断任务是否完成，采用多维度评估：

**覆盖度（Coverage）**
- 用户需求分解为子主题
- 每个子主题的信息覆盖度评估
- 整体覆盖度 = 已覆盖子主题数 / 总子主题数

**深度（Depth）**
- L1 提及：只提到概念
- L2 描述：有基本描述
- L3 分析：有数据支撑的分析
- L4 洞察：有深度洞察和预测

**可信度（Credibility）**
- 来源数量：单一来源 vs 多来源交叉验证
- 来源质量：官方/权威 vs 非官方/非权威
- 信息一致性：多来源是否一致

**时效性（Timeliness）**
- 关键数据的时间戳
- 行业变化速度
- 用户对时效的要求

### 6.2 终止判断策略

**目标驱动终止**
- 所有子主题覆盖度达到阈值
- 关键信息深度达标
- 信息来源可信度足够

**边际收益终止**
- 继续采集的边际收益低于阈值
- 连续多次搜索结果重复率高
- 新页面有效信息密度持续下降

**资源限制终止**
- 达到时间限制
- 达到页面访问限制
- 达到 Token 限制

**用户确认终止**
- 在不确定时主动询问用户
- 提供当前进度和未完成项

## 八、实现优先级

### 第一阶段：基础能力

- 基础理解能力：页面类型识别、简单结构分析
- 基础操作能力：导航、点击、输入、滚动
- 基础观察能力：变化检测、效果验证
- 会话记忆：操作历史、已访问 URL

### 第二阶段：增强能力

- 增强理解能力：页面布局模式识别
- 复合操作能力：翻页、搜索、筛选
- 异常识别：弹窗、验证码、错误提示
- 模式记忆：网站特征、操作技能

### 第三阶段：智能能力

- 智能规划：根据目标自动选择操作序列
- 完成度评估：多维度评估、终止判断
- 知识记忆：泛化到新网站
- OpenCLI 集成：生成确定性适配器

---

## 九、核心价值

1. **抽象层**：为上层应用提供统一的浏览器交互接口
2. **智能层**：提供理解、观察、记忆的智能能力
3. **学习层**：通过记忆积累，越来越擅长操作各种网站
4. **成本优化**：一次学习，多次零成本执行
