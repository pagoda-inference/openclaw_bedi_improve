# 报告生成能力设计

## 一、定位与目标

### 1.1 核心定位

报告生成能力是 OpenClaw 将调研数据和分析结果转化为结构化报告的能力模块。它承接数据处理模块的输出，生成用户可阅读、可分享、可演示的报告。

### 1.2 设计目标

- **智能适配**：根据调研内容自动选择合适的报告模板
- **灵活定制**：支持用户指定模板或自定义模板
- **HTML 优先**：以 HTML 为核心输出格式，支持后续转换
- **美观易读**：生成专业、美观、易读的报告

---

## 二、报告生成流程

### 2.1 整体流程

```
分析结果输入 → 模板选择 → 内容组织 → HTML 渲染 → 输出交付
```

### 2.2 流程详解

**模板选择**
- 用户指定模板 → 使用指定模板
- 用户未指定 → LLM 智能选择

**内容组织**
- 根据模板结构组织内容
- 填充数据到对应章节
- 生成图表和可视化

**HTML 渲染**
- 渲染 HTML 报告
- 应用样式和布局
- 添加交互功能（如导航）

**输出交付**
- 保存到任务输出目录
- 通过 Gateway 提供访问链接
- 支持下载

## 三、报告模板系统

### 3.1 模板分类

**按报告类型分类**

| 模板类型 | 适用场景 | 典型结构 |
|---------|---------|---------|
| 行业概览 | 市场规模、竞争格局、发展趋势 | 概述 → 市场分析 → 竞争格局 → 趋势预测 → 结论 |
| 竞品对比 | 产品对比、优劣势分析 | 概述 → 对比维度 → 对比表格 → 优劣势分析 → 选型建议 |
| 深度研究 | 背景分析、现状调研、前景展望 | 背景 → 现状 → 深度分析 → 展望 → 建议 |
| 趋势追踪 | 事件追踪、热度变化 | 概述 → 时间线 → 关键事件 → 趋势分析 → 预测 |
| 快速简报 | 快速了解、核心要点 | 核心结论 → 关键数据 → 主要发现 → 建议 |
| 数据报告 | 数据密集型分析 | 数据概览 → 详细数据 → 分析解读 → 附录 |

### 3.2 模板自动选择

当用户未指定模板时，系统使用 LLM 进行智能判断。

**LLM 判断方式**

将用户需求和数据摘要发送给 LLM，让 LLM 选择最合适的模板：

```
Prompt 示例：

用户需求：{user_input}
数据摘要：{data_summary}

可用模板：
A. 行业概览 - 适用于市场规模、竞争格局、发展趋势分析
B. 竞品对比 - 适用于产品/公司对比、优劣势分析
C. 深度研究 - 适用于深度主题调研、背景分析
D. 趋势追踪 - 适用于事件追踪、热度变化分析
E. 快速简报 - 适用于快速了解核心要点
F. 数据报告 - 适用于数据密集型分析

请选择最合适的模板，返回选项字母。
```

**判断依据**

LLM 会综合考虑：
- 用户原始需求的语义
- 调研数据的特征
- 各模板的适用场景

**优势**
- 理解语义，判断更准确
- 无需维护复杂规则
- 适应各种边界情况

### 3.3 模板结构定义

每个模板由以下部分组成：

**模板元信息**
```json
{
  "template_id": "industry_overview",
  "template_name": "行业概览",
  "description": "适用于市场规模、竞争格局、发展趋势分析",
  "keywords": ["市场", "行业", "规模", "竞争"],
  "output_formats": ["html", "pdf", "ppt", "markdown"]
}
```

**模板结构定义**
```json
{
  "sections": [
    {
      "id": "summary",
      "title": "核心概述",
      "required": true,
      "content_type": "text",
      "max_length": 500
    },
    {
      "id": "market_size",
      "title": "市场规模",
      "required": true,
      "content_type": "data",
      "suggested_charts": ["bar", "line"]
    },
    {
      "id": "competition",
      "title": "竞争格局",
      "required": true,
      "content_type": "analysis",
      "suggested_charts": ["pie", "table"]
    },
    {
      "id": "trends",
      "title": "发展趋势",
      "required": false,
      "content_type": "analysis"
    },
    {
      "id": "conclusion",
      "title": "结论与建议",
      "required": true,
      "content_type": "text"
    }
  ]
}
```

## 四、内容组织

### 4.1 内容映射

将分析结果映射到模板章节：

**自动映射规则**

| 数据类型 | 映射章节 |
|---------|---------|
| 市场规模数据 | 市场规模 |
| 竞争者信息 | 竞争格局 |
| 趋势数据 | 发展趋势 |
| 对比数据 | 对比分析 |
| 事件数据 | 时间线 |
| 统计数据 | 数据概览 |

**智能填充**
- 对于必填章节，确保有内容填充
- 对于可选章节，根据数据情况决定是否包含
- 对于缺失内容，标注"暂无数据"或省略该章节

### 4.2 图表生成

**图表类型选择**

| 数据特征 | 推荐图表 | 说明 |
|---------|---------|------|
| 数值对比 | 柱状图 | 不同对象的数值对比 |
| 时间变化 | 折线图 | 数值随时间变化 |
| 占比分布 | 饼图/环形图 | 各部分占比 |
| 多维度对比 | 雷达图 | 多个维度的对比 |
| 关联关系 | 散点图 | 两个变量的关系 |
| 层级结构 | 树图 | 层级数据展示 |
| 流程过程 | 流程图 | 步骤和流程 |

**图表生成方式**
- HTML 报告：使用 ECharts 或 Chart.js
- Markdown 报告：使用 Mermaid 语法
- PDF/PPT：使用预渲染的 SVG 或图片

### 4.3 文本生成

**章节内容生成**
- 核心概述：提炼关键发现，控制在 300-500 字
- 分析章节：结合数据和洞察，形成连贯叙述
- 结论建议：基于分析结果，给出明确结论和建议

**数据描述生成**
- 自动生成数据解读文字
- 突出关键数据和变化
- 保持客观、准确

## 五、HTML 报告渲染

### 5.1 基础结构

```html
<!DOCTYPE html>
<html>
<head>
  <meta charset="UTF-8">
  <title>{{report_title}}</title>
  <link rel="stylesheet" href="styles/report.css">
</head>
<body>
  <nav class="toc"><!-- 目录导航 --></nav>
  <main class="content">
    <!-- 报告内容 -->
  </main>
  <footer class="meta">
    <!-- 元信息：生成时间、数据来源等 -->
  </footer>
</body>
</html>
```

### 5.2 样式设计原则

- 响应式布局，适配不同设备
- 清晰的视觉层次
- 专业的配色方案
- 良好的可读性

### 5.3 交互功能

- 目录导航
- 返回顶部
- 图表交互（缩放、悬停提示）
- 打印优化

### 5.4 图表集成

使用 ECharts 或 Chart.js 嵌入交互式图表：

```html
<div class="chart-container">
  <div id="chart-market-size" style="width: 100%; height: 400px;"></div>
</div>
<script>
  // ECharts 配置
  var chart = echarts.init(document.getElementById('chart-market-size'));
  chart.setOption({
    // 图表配置
  });
</script>
```

## 六、模板库设计

### 6.1 内置模板

系统内置以下模板：

```
templates/
├── industry_overview/         # 行业概览
│   ├── template.json          # 模板定义
│   ├── template.html          # HTML 模板
│   └── styles.css             # 样式文件
│
├── competitor_compare/         # 竞品对比
│   ├── template.json
│   ├── template.html
│   └── styles.css
│
├── deep_research/             # 深度研究
│   ├── template.json
│   ├── template.html
│   └── styles.css
│
├── trend_tracking/            # 趋势追踪
│   ├── template.json
│   ├── template.html
│   └── styles.css
│
├── quick_brief/               # 快速简报
│   ├── template.json
│   ├── template.html
│   └── styles.css
│
└── data_report/               # 数据报告
    ├── template.json
    ├── template.html
    └── styles.css
```

### 6.2 自定义模板

用户可以创建自定义模板：

**模板目录结构**
```
custom_templates/
└── my_template/
    ├── template.json          # 必需：模板定义
    ├── template.html          # 必需：HTML 模板
    ├── styles.css             # 可选：自定义样式
    └── preview.png            # 可选：模板预览图
```

**模板定义规范**
```json
{
  "template_id": "my_template",
  "template_name": "我的自定义模板",
  "description": "模板描述",
  "author": "用户名",
  "version": "1.0.0",
  "sections": [
    {
      "id": "section_1",
      "title": "章节标题",
      "required": true,
      "content_type": "text"
    }
  ],
  "variables": [
    {
      "name": "report_title",
      "type": "string",
      "required": true
    }
  ]
}
```

### 6.3 模板配置

用户可以在任务配置中指定模板：

```json
{
  "report_template": "industry_overview",
  "report_formats": ["html", "pdf"],
  "report_style": {
    "primary_color": "#2563eb",
    "font_family": "system-ui",
    "include_toc": true,
    "include_sources": true
  }
}
```

## 八、报告质量保障

### 8.1 内容检查

**必填章节检查**
- 确保所有必填章节都有内容
- 缺失内容标注"暂无数据"

**数据一致性检查**
- 报告中的数据与分析结果一致
- 图表数据与文本描述一致

**格式规范检查**
- 标题层级正确
- 列表格式规范
- 表格对齐正确

### 8.2 视觉检查

**布局检查**
- 无明显空白区域
- 内容分布均匀
- 分页合理

**样式检查**
- 颜色搭配协调
- 字体大小合适
- 图表清晰可读

### 8.3 可读性检查

**文字可读性**
- 句子通顺
- 逻辑连贯
- 无明显错误

**数据可读性**
- 数值格式正确
- 单位标注清晰
- 图表标签完整

## 九、实现优先级

### 第一阶段：基础能力

| 功能 | 优先级 | 说明 |
|------|-------|------|
| 内置模板（3个） | P0 | 行业概览、竞品对比、快速简报 |
| HTML 输出 | P0 | 基础 HTML 报告 |
| LLM 模板选择 | P0 | LLM 智能选择模板 |
| 内容自动填充 | P0 | 数据映射到章节 |

### 第二阶段：增强能力

| 功能 | 优先级 | 说明 |
|------|-------|------|
| 图表生成 | P1 | 自动生成图表 |
| 自定义模板 | P1 | 支持用户自定义模板 |
| 内置模板（全部） | P1 | 补充剩余内置模板 |
| 样式优化 | P1 | 优化报告视觉效果 |

### 第三阶段：高级能力

| 功能 | 优先级 | 说明 |
|------|-------|------|
| PDF 转换 | P2 | HTML 转 PDF |
| PPT 转换 | P2 | 生成演示文稿 |
| 高级样式定制 | P2 | 用户自定义样式 |
| 报告版本管理 | P2 | 支持报告更新和版本对比 |

**内容组织**

**HTML 输出**
- 响应式布局
- 交互式图表
- 目录导航
- 打印优化

**质量保障**
- 内容完整性检查
- 数据一致性检查
