# OpenClaw 市场调研智能体能力需求

## 一、核心定位

OpenClaw 的目标是成为一个**适合市场调研-报告生成的智能体系统**。为此，需要在现有能力基础上增强以下核心能力：

1. **Deep Browser**：系统级别的浏览器深度操作能力
2. **智能规划与判断**：自主规划调研路径、判断完成度
3. **数据分析能力**：数据提取、验证、整合、可视化
4. **记忆与学习**：多层次记忆系统，越用越聪明

---

## 二、Deep Browser 核心能力

Deep Browser 是 OpenClaw 系统级别的浏览器能力增强模块，包含四大核心能力。

### 2.1 理解能力

**能力描述**：理解网页的布局结构、语义含义、交互方式和数据组织。

**具体要求**：

**页面类型识别**
- 识别常见页面类型：电商列表、博客文章、搜索结果、文档站、表格数据、新闻列表等
- 理解页面的主要目的：展示信息、收集数据、进行交易等

**页面结构理解**
- 识别页面布局模式：单栏、双栏、三栏、网格布局
- 识别页面元素：导航、侧边栏、内容区、页脚
- 定位主要内容区域，忽略广告、导航等干扰信息

**交互元素识别**
- 识别可交互元素：按钮、链接、表单、滚动区域
- 理解元素语义：搜索按钮、下一页、筛选器、展开按钮

**数据结构识别**
- 识别数据组织方式：表格、列表、卡片、JSON
- 理解数据之间的关系

### 2.2 操作能力

**能力描述**：对网页执行各种操作，从基础操作到复合操作。

**具体要求**：

**基础操作**
- 导航：打开 URL、前进、后退、刷新
- 点击：点击按钮、链接、复选框
- 输入：填写表单、输入文本
- 滚动：向上滚动、向下滚动、滚动到元素
- 等待：等待元素出现、等待页面加载、等待条件满足

**复合操作**
- 翻页：识别分页类型（数字分页、加载更多、无限滚动），执行翻页操作
- 搜索：定位搜索框，输入关键词，提交搜索
- 筛选：识别筛选器，选择筛选条件
- 展开：点击"查看更多"、"展开"等按钮
- 递进式探索：从入口页面开始，识别并访问相关链接

**策略操作**
- 根据目标选择最优操作序列
- 根据页面状态动态调整操作策略
- 处理异常情况（弹窗、验证码、错误提示）

### 2.3 观察能力

**能力描述**：每次操作后观察效果，形成反馈循环。

**具体要求**：

**变化检测**
- 检测页面是否发生变化
- 识别变化类型：URL 变化、内容变化、状态变化
- 判断变化是否符合预期

**效果验证**
- 验证操作是否成功
- 判断是否达到预期目标
- 检测是否产生副作用

**状态感知**
- 感知当前状态：正常、加载中、处理中、错误
- 识别中间状态和最终状态

**异常识别**
- 识别弹窗、验证码、错误提示
- 识别登录需求、权限限制
- 识别页面崩溃、网络错误

### 2.4 记忆能力

**能力描述**：积累操作经验，让系统越用越聪明。

**具体要求**：

**会话记忆**
- 记录已访问的 URL，避免重复访问
- 记录已获取的数据片段
- 记录当前操作状态，支持任务恢复

**模式记忆**
- 记录网站布局模式
- 记录操作技能模板
- 记录成功/失败经验

**知识记忆**
- 记录网站特征知识
- 记录数据结构知识
- 记录最佳实践

## 三、智能规划与判断能力

### 3.1 调研规划能力

**能力描述**：根据用户需求，自主规划调研路径。

**具体要求**：

**需求分析**
- 分析用户输入，提取调研主题
- 识别调研类型：关键词调研、指定网站分析、混合输入
- 判断调研深度：快速了解、标准调研、深度研究

**计划制定**
- 分解调研主题为子主题
- 制定搜索关键词列表
- 确定网站抓取顺序
- 设定信息优先级

**动态调整**
- 根据已获取信息调整搜索关键词
- 根据信息质量调整抓取深度
- 根据发现的新信息源调整抓取顺序

### 3.2 完成度评估能力

**能力描述**：自主判断调研是否完成，何时可以停止。

**具体要求**：

**多维度评估**

| 维度 | 评估内容 | 指标 |
|------|---------|------|
| 覆盖度 | 子主题信息覆盖情况 | 已覆盖子主题数 / 总子主题数 |
| 深度 | 信息详细程度 | L1提及 → L4洞察 |
| 可信度 | 信息来源质量 | 来源数量、来源权威性、一致性 |
| 时效性 | 信息新鲜度 | 数据时间戳、行业变化速度 |

**终止判断策略**

- **目标驱动终止**：覆盖度、深度、可信度均达标
- **边际收益终止**：继续采集的边际收益低于阈值
- **资源限制终止**：达到时间或资源限制
- **用户确认终止**：不确定时询问用户

### 3.3 自我评估机制

**能力描述**：阶段性自检，确保调研质量。

**具体要求**：

**阶段性自检**
- 信息缺口检查：当前信息能否回答用户问题
- 逻辑一致性检查：不同来源信息是否存在矛盾
- 深度评估：当前信息是表面描述还是有数据支撑
- 来源多样性检查：是否过度依赖单一来源

**报告生成前终检**
- 是否回答了用户的核心问题
- 关键数据是否有来源支撑
- 是否存在明显的信息缺口
- 信息是否足够新鲜
- 是否存在未解决的矛盾

## 四、数据分析能力

### 4.1 数据提取能力

**能力描述**：从网页中提取结构化数据。

**具体要求**：

**结构化数据提取**
- 表格数据识别和提取
- 列表数据提取（产品列表、功能列表）
- 键值对提取（参数、规格）
- 时间线/事件序列提取

**数值数据提取**
- 金额识别（货币符号、单位转换）
- 百分比和比例
- 时间和日期
- 数量级识别（万、亿、M、B）

**关系数据提取**
- 实体关系（公司-产品-人物）
- 层级关系（分类、组织架构）
- 时间关系（先后、因果）
- 对比关系（优劣势、异同）

### 4.2 数据验证能力

**能力描述**：验证数据的准确性和一致性。

**具体要求**：

**交叉验证**
- 同一数据点从多个来源验证
- 识别数据冲突
- 标注数据可信度

**逻辑验证**
- 总量 = 各部分之和验证
- 增长率计算验证
- 时间线合理性验证

**异常检测**
- 识别异常大/异常小的数值
- 识别与已知事实矛盾的数据
- 识别格式异常（可能是 OCR 错误）

### 4.3 数据整合能力

**能力描述**：整合来自多个来源的数据。

**具体要求**：

**数据标准化**
- 货币统一：$100M / 1亿美元 / 100 million USD
- 时间统一：2024年Q1 / Q1 2024 / 第一季度
- 单位统一：万/亿/M/B
- 名称统一：公司名、产品名的不同写法

**数据聚合**
- 时间序列聚合：不同时间点的数据形成趋势
- 分类聚合：按类别汇总数据
- 地域聚合：按地区汇总数据

**数据补全**
- 基于增长率推算未来值
- 基于占比推算总量
- 明确标注为"估算值"

### 4.4 数据可视化能力

**能力描述**：将数据转换为可视化图表。

**具体要求**：

**图表类型选择**
- 对比数据 → 柱状图/条形图
- 趋势数据 → 折线图/面积图
- 占比数据 → 饼图/环形图
- 关系数据 → 关系图/树图

**图表生成**
- 使用 Mermaid 语法（Markdown 支持）
- 使用 ECharts/Chart.js（HTML 支持）
- 生成 SVG 静态图表

## 五、报告生成能力

### 5.1 报告结构设计能力

**能力描述**：根据调研主题设计合适的报告结构。

**具体要求**：

**报告类型识别**
- 行业概览：市场规模、竞争格局、发展趋势
- 竞品对比：功能对比、优劣势分析、选型建议
- 深度研究：背景分析、现状调研、前景展望
- 趋势追踪：重要事件、热度变化、新兴话题

**结构设计**
- 章节划分
- 内容层次安排
- 逻辑连贯性保证

### 5.2 报告布局能力

**能力描述**：设计报告的视觉布局。

**具体要求**：

**页面布局**
- 标题层级设计
- 段落间距设置
- 表格和图表位置安排
- 引用和注释格式

**样式设计**
- 字体选择
- 颜色搭配
- 响应式适配

### 5.3 格式转换能力

**能力描述**：将报告转换为多种格式。

**具体要求**：

- HTML 生成：交互式报告，支持导航、搜索
- PDF 生成：适合打印和分享
- PPT 生成：适合演示，自动拆分章节
- Markdown 生成：适合技术文档
- 数据导出：CSV、JSON 格式

## 七、能力发展路径

### 第一阶段：基础能力（MVP）

- 基础理解能力：页面类型识别、简单结构分析
- 基础操作能力：导航、点击、输入、滚动
- 基础规划能力：简单关键词规划
- 基础数据提取：文本提取、表格提取
- 基础报告生成：单一模板、基础布局
- 会话记忆：操作历史、已访问 URL

### 第二阶段：增强能力

- 增强理解能力：页面布局模式识别
- 复合操作能力：翻页、搜索、筛选
- 智能规划调整：动态调整策略
- 完成度评估：多维度评估、终止判断
- 数据验证：交叉验证、异常检测
- 模式记忆：网站特征、操作技能

### 第三阶段：智能能力

- 智能理解：自动发现网站数据结构
- 策略操作：根据目标自动选择操作序列
- 自我评估：阶段性自检、报告前终检
- 数据可视化：自动图表生成
- 知识记忆：泛化到新网站
- OpenCLI 集成：生成确定性适配器

## 八、与 OpenCLI 的关系

OpenCLI 提供了"编译期智能 vs 运行期智能"的设计思路：

- **一次生成，多次零成本执行**：首次访问网站时用 LLM 分析，生成模式描述；后续访问直接使用模式，零 LLM 成本
- **确定性适配器**：对于频繁访问的网站，生成 OpenCLI 适配器，完全确定性执行
- **技能沉淀**：将成功的操作序列抽象为技能模板，复用经验

Deep Browser 将借鉴这一思路，实现成本优化和稳定性提升。
