# 页面分析参考文档

本文档提供页面结构分析的方法论和框架，帮助智能体理解网页结构并发现导航入口。

## 分析目标

页面分析的核心目标是理解：
1. **页面用途** - 这个页面是做什么的
2. **内容结构** - 信息如何组织
3. **导航入口** - 从这个页面可以到达哪些其他页面（关键）
4. **交互方式** - 哪些交互可以发现隐藏内容
5. **安全性** - 哪些交互是只读的，哪些是写入的

## 分析维度

### 1. 页面类型识别

**常见页面类型：**

| 类型 | 特征 | 导航价值 | 示例 |
|------|------|---------|------|
| listing | 列表展示，通常有分页 | ⭐⭐⭐ 高 | 商品列表、文章列表 |
| product | 单个实体详情 | ⭐⭐ 中 | 商品详情页、文章详情 |
| search | 搜索结果展示 | ⭐⭐⭐ 高 | 搜索结果页 |
| login | 用户认证 | ⭐ 低（写入类） | 登录、注册页面 |
| form | 数据收集 | ⭐ 低（写入类） | 联系表单、调查问卷 |
| content | 信息展示 | ⭐ 低 | 关于我们、帮助文档 |
| checkout | 交易流程 | ❌ 禁止（写入类） | 购物车、结算页 |
| dashboard | 数据概览 | ⭐⭐ 中 | 用户中心、控制面板 |

**识别方法：**
- 查看页面标题和主要标题（h1）
- 观察主要内容区域的布局
- 识别关键交互元素（按钮、表单）
- 分析URL路径和参数

### 2. 导航入口发现（核心维度）

每个页面的首要分析任务是发现所有导航入口，即从当前页面可以到达的其他页面。

**导航入口类型：**

| 入口类型 | 发现方式 | 导航价值 | 安全性 |
|---------|---------|---------|--------|
| 导航链接 | 直接可见 | ⭐⭐⭐ | ✅ 只读 |
| 面包屑 | 直接可见 | ⭐⭐⭐ | ✅ 只读 |
| 列表项链接 | 直接可见 | ⭐⭐⭐ | ✅ 只读 |
| 分页链接 | 直接可见 | ⭐⭐⭐ | ✅ 只读 |
| 折叠菜单中的链接 | 需展开 | ⭐⭐ | ✅ 只读 |
| 标签切换中的链接 | 需切换 | ⭐⭐ | ✅ 只读 |
| 下拉菜单中的链接 | 需触发 | ⭐⭐ | ✅ 只读 |
| 搜索结果中的链接 | 需搜索 | ⭐⭐⭐ | ✅ 只读 |
| 滚动加载的条目 | 需滚动 | ⭐⭐ | ✅ 只读 |
| 页脚链接 | 直接可见 | ⭐⭐ | ✅ 只读 |

**发现方法：**
1. 扫描所有 `<a>` 标签，记录 href 属性
2. 扫描所有导航区域（nav, header, footer）
3. 检查折叠/隐藏区域中是否有链接
4. 执行交互（展开、滚动）后重新扫描
5. 记录每个入口的目标URL和描述

### 3. 布局模式分析

**常见布局模式：**

```
单列布局 (single-column)
┌─────────────┐
│   Header    │
├─────────────┤
│   Content   │
│             │
├─────────────┤
│   Footer    │
└─────────────┘

双列布局 (two-column)
┌─────────────┐
│   Header    │
├──────┬──────┤
│ Main │ Side │
│      │ bar  │
├──────┴──────┤
│   Footer    │
└─────────────┘

三列布局 (three-column)
┌─────────────┐
│   Header    │
├──┬────┬─────┤
│L │Main│  R  │
│  │    │     │
├──┴────┴─────┤
│   Footer    │
└─────────────┘
```

**分析方法：**
- 使用浏览器开发工具查看DOM结构
- 识别主要容器元素（div, section, article）
- 观察CSS类名（container, wrapper, sidebar等）
- 检查响应式断点

### 4. 数据区域定位

**数据区域特征：**

| 数据类型 | HTML特征 | CSS选择器示例 |
|---------|---------|--------------|
| 商品列表 | ul/ol, div.grid | .product-list, .products |
| 文章列表 | article, div.post | .article-list, .posts |
| 表格数据 | table | table, .data-table |
| 表单 | form | form, .contact-form |
| 导航 | nav, ul.menu | nav, .navigation |

**定位方法：**
1. 查找重复元素（列表项、卡片）
2. 识别数据容器
3. 确定数据字段（标题、价格、描述等）
4. 验证选择器稳定性

### 5. 分页机制识别

**分页类型：**

| 类型 | 特征 | 安全性 | 识别方法 |
|------|------|--------|---------|
| click | 页码按钮、上一页/下一页 | ✅ 只读 | 查找分页导航元素 |
| scroll | 滚动加载更多 | ✅ 只读 | 监听滚动事件，观察内容变化 |
| load-more | "加载更多"按钮 | ✅ 只读 | 查找按钮文本或图标 |
| none | 单页内容 | - | 无分页元素 |

**识别步骤：**
1. 查找分页相关元素
2. 检查是否有"下一页"、"加载更多"等文本
3. 测试交互行为
4. 记录选择器和交互方式

### 6. 表单分析

**表单分类（关键：区分读取类和写入类）：**

| 表单类型 | 用途 | 安全性 | 处理方式 |
|---------|------|--------|---------|
| 搜索表单 | 获取搜索结果 | ✅ 只读 | 允许输入和提交 |
| 筛选表单 | 改变展示方式 | ✅ 只读 | 允许选择和提交 |
| 登录表单 | 创建会话 | ❌ 写入 | 仅记录入口 |
| 注册表单 | 创建账号 | ❌ 写入 | 仅记录入口 |
| 联系表单 | 发送消息 | ❌ 写入 | 仅记录入口 |
| 订单表单 | 产生交易 | ❌ 写入 | 仅记录入口 |

**分析要点：**
- 表单用途（搜索、登录、联系等）
- 表单安全性分类（读取类 vs 写入类）
- 必填字段识别（required属性、*标记）
- 输入类型（text、email、password等）
- 提交方式（按钮、回车等）

### 7. 导航路径发现

**导航类型：**

| 导航类型 | 位置 | 用途 | 安全性 |
|---------|------|------|--------|
| 主导航 | 页面顶部 | 主要页面切换 | ✅ 只读 |
| 侧边导航 | 页面左侧 | 分类、筛选 | ✅ 只读 |
| 面包屑 | 内容顶部 | 层级导航 | ✅ 只读 |
| 页脚导航 | 页面底部 | 辅助链接 | ✅ 只读 |

**发现方法：**
- 查找nav元素
- 识别链接文本和目标
- 分析导航层级关系
- 记录重要链接

## 分析工具

### 浏览器开发工具

**元素检查：**
```javascript
document.querySelector('selector')
element.getAttribute('class')
element.textContent
```

**导航入口发现：**
```javascript
// 查找所有链接
document.querySelectorAll('a')

// 查找导航区域
document.querySelectorAll('nav a')

// 查找面包屑
document.querySelectorAll('.breadcrumb a')

// 查找分页
document.querySelectorAll('.pagination a')
```

**网络监控：**
- 打开Network面板
- 筛选XHR/Fetch请求
- 查看请求参数和响应

### 选择器测试

**CSS选择器：**
```javascript
document.querySelectorAll('.product-item')
document.querySelectorAll('.product-item').length
```

**XPath：**
```javascript
document.evaluate(
  '//div[@class="product"]',
  document,
  null,
  XPathResult.ANY_TYPE,
  null
)
```

| .next-page | /products?page=2 | 下一页 | ✅ 只读 |

| 选择器 | 类型 | 描述 |

## 表单
| 用途 | 选择器 | 字段 | 安全性 |
|------|--------|------|--------|

## 最佳实践

1. **优先发现导航入口**
   - 每个页面的首要任务是发现所有导航入口
   - 记录每个入口指向的URL
   - 逐层深入探索

2. **从整体到局部**
   - 先识别页面类型和布局
   - 再定位导航入口和数据区域
   - 最后分析细节元素

3. **安全性判断**
   - 对每个交互元素判断只读/写入
   - 表单必须分类为读取类或写入类
   - 无法确定时视为写入类

4. **验证选择器**
   - 使用开发工具测试
   - 检查选择器稳定性
   - 考虑动态内容影响

5. **记录发现**
   - 及时记录分析结果
   - 更新状态文件
   - 标注不确定项

6. **迭代优化**
   - 初步分析后验证
   - 根据实际情况调整
   - 持续完善分析结果

## 常见问题

**Q: 如何处理动态加载的内容？**
A: 等待内容加载完成，或触发加载事件后再分析。动态加载的内容可能包含新的导航入口。

**Q: 如何识别反爬机制？**
A: 观察请求头要求、验证码、登录墙等

**Q: 选择器不稳定怎么办？**
A: 使用多个备选选择器，或基于内容特征定位

**Q: 如何确保不遗漏导航入口？**
A: 系统化扫描所有区域（导航栏、内容区、侧边栏、页脚），执行所有安全交互后重新扫描
